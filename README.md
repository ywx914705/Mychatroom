Mychatroom — 基于 Reactor 模型的自实现高性能聊天室
📌 项目概述
Mychatroom 是一个基于 C++ 编写的简易聊天室项目，采用 Reactor 模型 与 epoll 事件驱动机制 实现高并发网络通信。该项目从底层网络通信模块到上层任务调度均采用自实现组件，包括线程池及其内部任务队列（自实现链表），具备良好的学习与扩展价值。

🛠 技术架构
核心模型：Reactor + epoll
采用 Reactor 设计模式，实现事件分发与业务逻辑解耦。

使用 epoll 边缘触发（ET）模式 实现高并发 I/O 多路复用，支持大量客户端连接。

自实现事件循环（EventLoop），统一管理连接、读、写等事件。

线程池与任务队列
自实现线程池，支持动态线程管理与任务调度。

线程池内部任务队列使用 自实现的链表（List） 结构，避免依赖 STL，提升可控性与性能。

任务队列支持线程安全的任务投递与获取。

 网络模块
基于 TCP 协议实现稳定可靠的双向通信。

支持多客户端同时在线、消息广播、用户上下线通知等功能。

使用非阻塞 socket 配合 epoll，避免线程阻塞，提升系统吞吐量。

数据结构
自实现链表（List）用于线程池任务队列，支持高效插入与删除。

使用哈希表（或自实现映射）管理客户端连接会话。

🚀 功能特点
支持多用户同时在线聊天

实时广播消息给所有在线用户

用户连接/断开通知

基于事件驱动的高并发处理

自实现线程池与任务队列，无 STL 依赖

代码结构清晰，模块化设计，便于扩展

🧪 编译与运行
环境要求
Linux 系统（支持 epoll）

g++ 支持 C++11 或以上

编译指令
bash
make clean
make
分别启动服务器与客户端
服务器只需要./程序名+port即可，而客户端要有ip以及port

bash
telnet 127.0.0.1 8888
📈 性能特点
使用 epoll ET 模式，减少系统调用次数。

自实现线程池避免频繁创建销毁线程。

自实现链表队列轻量高效，无额外依赖。
